# 剧本杀单人本创作指南 - Hermes 使用文档

> **目标**: 生成符合系统规范的单人剧本杀JSON文件，风格参考《问佛》，沉浸式解谜体验。

---

## 📋 核心定位

**单人本 = 沉浸式解谜 + 关键词探索 + 记忆拼图**

- 玩家扮演**唯一角色**，通过探索关键词逐步揭开真相
- **NPC列表**代替多角色，玩家与NPC互动推进剧情
- **关键词系统**：地点、物品、人物、回忆等，输入关键词触发剧情
- **记忆碎片**：通过特定物品+回忆触发失去的记忆

---

## 🎯 单人本 vs 多人本

| 项目 | 多人本 | 单人本 |
|------|--------|--------|
| 玩家数量 | 2-4人 | 1人 |
| 角色设定 | 多个可选角色 | 唯一主角 |
| NPC | 无或少量 | 5-8个NPC |
| 交互方式 | 玩家间讨论 | 关键词探索 |
| 线索获取 | 轮次释放 | 关键词触发 |
| 游戏时长 | 5-12分钟 | 6-8分钟 |
| 剧情深度 | 推理为主 | 沉浸式叙事 |

## 📝 JSON结构调整

### 关键差异

```json
{
  "player": 1,  ← 固定为1
  "scriptId": 900010,  ← 建议从900010开始
  "style": "沉浸式解谜 / 关键词探索 / 记忆拼图",
  "characters": [
    {
      "id": 1001,  ← 唯一主角
      "name": "主角名",
      "data": {
        "script": "你是XX。失去了一年的记忆...",
        "task": "任务：\n1、找回记忆\n2、解决疑点\n3、揭开真相",
        "npc_list": [  ← 新增NPC列表
{
  "script": {
    "scriptId": 900010,
    "scriptName": "剧本名称",
    "status": 1,  ← 新增：剧本状态（1=显示，0=隐藏，默认为1）
    ...
  }
}
    }
  ],
  "rounds": []  ← 单人本理论上不使用轮次系统，但为兼容前端，建议仍然添加 rounds 数据
}
```

**⚠️ 重要更新（2026/6/25）**：
虽然单人本的设计理念是关键词探索，但当前前端代码依赖 `rounds` 数组来渲染游戏流程。因此：
- **建议做法**：根据 `flow` 字段创建对应的 `rounds` 数组
- **`round` 字段**：必须等于 `rounds` 数组长度
- **未来优化**：可以开发专门的单人本关键词探索界面

## 🎨 创作核心

### 1. 关键词系统（核心玩法）

**关键词类型**：
- 🏠 **地点关键词**：【香积寺】【青川城】【梵音寺】
- 📦 **物品关键词**：【莲花玉坠】【染血佛珠】【焦尾琴】
- 👤 **人物关键词**：【尹念君】【步莲】【疯道人】
- 💭 **回忆关键词**：【物品名+回忆】如【染血佛珠回忆】
- 🔗 **组合关键词**：【如意坊+银镯】需两个关键词组合
- 🎁 **彩蛋关键词**：隐藏线索，找到可降低推理难度

**关键词数量建议**：
- 地点：15-20个
- 物品：5-8个
- 人物：5-8个
- 回忆：5-7个
- 组合：2-3个
- 彩蛋：1-2个

### 2. 记忆拼图机制

```
主角失去记忆 
→ 探索关键词获得物品线索
→ 输入【物品名+回忆】
→ 触发记忆碎片
→ 拼凑完整真相
```

### 3. NPC互动

```json
"npc_list": [
  {
    "name": "神秘道人",
    "introduce": "蓬头垢面，神神叨叨",
    "relation": "指引者",
    "keyword_trigger": "【疯道人】"
  }
]
```

## 📤 题材建议

### ✅ 适合的题材（参考《问佛》）

- 🏯 **古风悬疑**：寺庙、宫廷、江湖
- 💔 **情感向**：前尘往事、爱而不得
- 🔮 **奇幻元素**：妖、鬼、神、佛
- 🧩 **失忆题材**：找回记忆的过程
- 📖 **文学性**：诗词、典故、禅理

### 剧情框架模板

**起**：主角失去记忆，踏上寻找真相之路  
**承**：探索关键词，遇见NPC，获得线索  
**转**：记忆逐渐恢复，真相呼之欲出  
**合**：最终选择，面对真相
