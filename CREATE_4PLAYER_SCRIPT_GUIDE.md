## ⚠️ 重要：round 字段说明

**关键规则**：
- `round` 字段的值 **必须等于** `rounds` 数组的长度
- 前端通过 `round` 或 `rounds.length` 判断游戏有多少轮
- `flow` 字段中的描述（如"三轮搜证"）只是说明性文字，实际轮次由 `round` 决定

**示例**：
```json
{
  "round": 3,           ← 表示有3轮
  "rounds": [           ← 必须包含3个元素
    { "round": 1, ... },
    { "round": 2, ... },
    { "round": 3, ... }
  ]
}
```

**常见错误**：
- ❌ `round: 3` 但 `rounds` 只有 2 个元素
- ❌ `flow` 写"三轮搜证"但 `round: 4`
- ✅ `round: 4` 且 `rounds` 有 4 个元素（正确）

## 🎯 4人本 vs 2人本 vs 3人本

| 项目 | 2人本 | 3人本 | 4人本 |
|------|------|------|------|
| 玩家数量 | 2人 | 3人 | 4人 |
| 角色ID | 4001-4002 | 3001-3003 | 4001-4004 |
| 角色关系 | 双人对抗/合作 | 三角关系 | 四方博弈/两两联盟 |
| 线索数量 | 每轮1-2条 | 每轮2条 | 每轮2-3条 |
| 讨论问题 | 每轮1-2个 | 每轮2个 | 每轮2-3个 |
| 游戏时长 | 4-7分钟 | 5-8分钟 | 8-12分钟 |
| 推理难度 | 简单 | 中等 | 中高 |

## 📝 关键差异

### 1. player 字段
```json
"player": 4  ← 改为4
```

### 2. scriptId 和 status
```json
"scriptId": 900003,  ← 建议从900003开始
"status": 1          ← 新增：1=显示（默认），0=隐藏
```

### 3. 4个角色（ID: 4001, 4002, 4003, 4004）
```json
"characters": [
  {"id": 4001, "name": "角色A", "voice": "白桦", "flag": 1, ...},
  {"id": 4002, "name": "角色B", "voice": "茉莉", "flag": 2, ...},
  {"id": 4003, "name": "角色C", "voice": "苏打", "flag": 4, ...},
  {"id": 4004, "name": "角色D", "voice": "冰糖", "flag": 8, ...}
]
```

**注意**: `flag` 字段使用位标志：1, 2, 4, 8

### 4. 音频路径
```
/audio/[剧本英文名]/
├── characters/
│   ├── 4001/  ← 角色A
│   ├── 4002/  ← 角色B
│   ├── 4003/  ← 角色C（新增）
│   └── 4004/  ← 角色D（新增）
└── rounds/
    ├── 1/
    ├── 2/
    └── 3/
```

## 🎤 4人本音色搭配建议

### 推荐配置（多样化）
- 👨‍💼 角色A：**白桦**（成熟男性）
- 👩‍💼 角色B：**茉莉**（知性女性）
- 👨 角色C：**苏打**（理性男性）
- 👧 角色D：**冰糖**（年轻女性）

### 其他搭配
- **全男性**：白桦 + 苏打 + 白桦 + 苏打（混用）
- **全女性**：茉莉 + 冰糖 + 茉莉 + 冰糖（混用）
- **性别平衡**：2男2女

## 🎨 4人本题材建议

### ✅ 适合的题材
- 🏢 **公司政治**：4位高管的权力斗争
- 🏰 **古风权谋**：皇子、大臣、宫女、侍卫
- 🏠 **豪门遗产**：4位继承人的争夺
- 🔐 **密室逃脱**：4位被困者揭开真相
- 🎭 **身份迷局**：每人都隐藏身份

### ❌ 避免的题材

- 需要5人以上才能平衡的复杂博弈

```
请按照以下要求创作一个4人剧本杀：

【剧本设定】
- 题材：[如"公司权力斗争"]
- 风格：中度推理 / 角色扮演
- 时长：用户自定义
- 人数：4人

【项目位置 - VPS环境】
- 项目根目录：/home/back/script_kill
- 输出文件到：/home/back/script_kill/demo/
- 文件名格式：demo/[剧本英文名]_4player.json
- 参考文件：/home/back/script_kill/demo/yizhu_weimian_demo.json

【要求】
1. 严格参考 demo/yizhu_weimian_demo.json 文件结构
2. 参考 TTS_CONFIG.md 选择音色
3. 使用默认封面 `/default_cover.png`
4. 使用默认音乐 `/sleep.mp3`
5. scriptId 使用 900003（如已存在则递增）
6. player 字段固定为 4
7. 4个角色，ID为 4001-4004

【角色要求】
- 4个角色，ID为 4001, 4002, 4003, 4004
- flag字段分别为 1, 2, 4, 8
- 音色配置：建议白桦、茉莉、苏打、冰糖
- 每个角色至少2条私密线索

【轮次要求】
- 建议3轮搜证（可根据剧情调整为4轮或更多）
- round 字段值必须等于 rounds 数组长度
- 每轮2-3条公开线索
- 每轮2-3个讨论问题

【真相设计】
- 4人需要拼凑各自线索才能还原完整真相
- 至少3个最终问题
- 提供完整的真相揭示

【音频路径规范】
- 音频根目录：/audio/[剧本英文名]/
- 角色音频：/audio/[剧本英文名]/characters/4001-4004/
- 轮次音频：/audio/[剧本英文名]/rounds/1/

请开始创作！
```
